# Linux 커널 시스템 콜 기반 오프라인 샌드박스 저지


## 1. 프로젝트 소개
본 프로젝트는 ELEC462 시스템 프로그래밍 과제로 개발된 오프라인 알고리즘 채점 시스템입니다.
C, C++, Python3 언어를 지원하며, 제출된 코드의 무한 루프, 메모리 초과, 악의적인 네트워크 접근 등으로부터 호스트 서버를 완벽하게 보호하기 위해 Linux 커널 레벨의 샌드박싱(seccomp, setrlimit)을 적용했습니다. 또한 터미널 환경에서 방향키로 반례를 탐색할 수 있는 직관적인 대화형 TUI 뷰어를 제공합니다.

## 2. 주요 핵심 기능
1. **커널 레벨 샌드박스 격리**: `seccomp` 필터를 통한 네트워크 통신(`socket`, `connect` 등) 원천 차단 및 `setrlimit`을 활용한 프로세스 자원(CPU, 메모리, NPROC) 강제 제한.
2. **비동기 I/O 멀티플렉싱**: 파이프 버퍼 용량 초과로 인한 데드락을 방지하기 위해 `poll` 시스템 콜 기반 비동기 데이터 소거 기능 구현.
3. **실시간 워치독(Watchdog) 타이머**: `pthread` 스레드를 활용하여 시간 초과(TLE) 프로세스 실시간 감지 및 강제 종료(`SIGKILL`).
4. **대화형 TUI 뷰어**: `termios` 원시 모드(Raw Mode)와 `fcntl` 논블로킹 제어를 통해 방향키로 오답 반례를 직관적으로 탐색.

## 3. 사용 방법

### 3.1. 빌드 및 실행
```bash
# 빌드
make

# 프로그램 실행
./judge
```
수동 컴파일 : gcc -o judge judge.c -lpthread -lseccomp -Wall

### 3.2. 필수 디렉토리 구조
```
./
├── judge             # 컴파일된 본 채점기 실행 파일
├── test.c            # 채점할 타깃 소스 코드 (제출 파일)
└── 1005/             # 문제 번호 폴더 (예: 1005번)
    ├── Correct/
    │   └── sol       # 정답 기준 실행 파일 (사전 컴파일 필수)
    ├── Generators/
    │   └── gen       # 입력 데이터 생성기 실행 파일 (사전 컴파일 필수)
    ├── Testcase/     # (실행 시 테스트케이스 .txt 파일이 임시 입출력되는 폴더)
    └── config.txt    # (선택) 시간(s) 및 메모리(MB) 제한 설정 파일
```

### 3.3. 환경 의존성
본 프로그램을 빌드하고 실행하기 위해서는 libseccomp 라이브러리가 필요합니다.
```
sudo apt-get update
sudo apt-get install libseccomp-dev
```
